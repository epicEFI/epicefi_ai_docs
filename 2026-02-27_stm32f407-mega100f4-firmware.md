# STM32F407 (STMpro Speeduino) Firmware Compatibility with epicEFI / Mega100F4

*Source: epicEFI Discord, 2026-02-27 | Channel: 1356732771325968630*
*Contributors: @MykkH*

## Summary

A user asked whether the **Mega100F4** firmware is the correct recommendation for an **STMpro Speeduino** board based on the STM32F407, and whether the STM32F407 can accept epicEFI / rusEFI firmware. The STMpro ordering site lists an option to run rusEFI with a different module, which caused some confusion.

## Details

### Hardware in Question

- **Board**: STMpro Speeduino (third-party Speeduino-compatible board using STM32F407 MCU)
- **Question**: Can the STM32F407 on this board run epicEFI/rusEFI firmware? And is Mega100F4 the correct firmware image?

### Key Points

- The STM32F407 MCU is supported by rusEFI/epicEFI firmware.
- The **Mega100F4** firmware image targets STM32F407-based boards in the Arduino Mega 2560 footprint (100-pin F4).
- The STMpro Speeduino uses the STM32F407, so the Mega100F4 image is the appropriate firmware to try — **subject to verifying that the pin mapping matches the specific board layout**.
- The STMPro ordering site mention of "run rus with a different module" likely refers to an optional H7-based module variant, not the base F407 board.

### Cautions

- Before flashing: confirm that the Mega100F4 pin assignments match the STMpro Speeduino's physical connectors, particularly for:
  - Trigger inputs (crank/cam)
  - Injector outputs
  - Ignition outputs
  - Analog sensor inputs (CLT, IAT, MAP, TPS)
- If pin mappings differ, a custom board definition YAML will be needed (see analog input YAML config documentation).
- The STMpro Speeduino may have been designed for Speeduino firmware, which has different pin assignments than rusEFI/epicEFI.

### Related

- For custom board pin configuration, see the YAML-based board definition workflow described in the analog input configuration documentation.
- The dcwerx STM32-H7 adapter board (`https://dcwerxtuned.com/product/stm32-h7-adapter-board-arduino-mega-2560-footprint/`) is a first-party epicEFI board in the same Arduino Mega 2560 form factor if a fully-compatible board is preferred.
